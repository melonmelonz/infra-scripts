Server Network Setup — Quick Guide

No Linux knowledge needed. Two cables, five typed lines, one photo. About 10 minutes.

Today you only need the two bold rows: internet feed into ether1, server into ether8. Everything else can be plugged in any time.

Steps at the server

1
Plug in the two cables — home router → ether1; server's 10G port → ether8. The server has two identical 10G ports — if step 3's first ping fails, move the cable to the other one.
2
Log in at the server — plug a monitor + keyboard into the server. Username root, password is the one set during the Proxmox install.
3
Type these four lines (press Enter after each; straight quotes, exact spacing). Each command is ONE line — if it wraps on this page, keep typing on the same line: 
sed -i 's|address .*|address 10.10.10.10/24|; s|gateway .*|gateway 10.10.10.1|' /etc/network/interfaces

sed -i '/bridge-fd/a\ bridge-vlan-aware yes\n bridge-vids 2-4094' /etc/network/interfaces

ifreload -a

ping -c 2 10.10.10.1 && ping -c 2 1.1.1.1
You want replies from both pings:
• 10.10.10.1 replies = server sees the router. If not → check the ether8 cable / try the other 10G port.
• 1.1.1.1 replies = internet works. If not → check the ether1 cable.
4
Type this one line — it sets up remote access and prints a hardware report: 
curl -fsSL https://raw.githubusercontent.com/melonmelonz/infra-scripts/main/scripts/host-bootstrap.sh | bash
It should end with BOOTSTRAP DONE.
5
Take a photo of everything from === IP === down and send it to Penn. That's it — the rest happens remotely.
If the report's === IOMMU === section is empty: reboot, press Del to enter BIOS, make sure SVM and IOMMU are both Enabled (Advanced → CPU / AMD CBS), boot back up, redo step 4.
